Welcome to Pyblue!

A simple static site generator. No configuration required. It just works on any existing site as well.

This file is located in doc/src/index.html of the PyBlue distribution.

  1. Launch pyblue serve -f doc/src
  2. View your site by visiting http:://localhost:8080.
The pages are rendered through the Django Template engine. Users automatically get additional context for each of the pages. Examples:
  • Writing: {% verbatim %}{{ p.name }}{% endverbatim %} will produce: {{ p.name }}
  • Writing: {% verbatim %} {{ p.created_date }} {% endverbatim %} will produce: {{ p.created_date }}
  • Writing: {% verbatim %} {{ p.last_modified }} {% endverbatim %} will produce: {{ p.last_modified }}
  • Writing: {% verbatim %} {{ p.size|filesizeformat }} {% endverbatim %} will produce: {{ p.size|filesizeformat }}

Note how in this last case we also made use of the filesizeformat builtin filter from Django.

There are many other additional context variables. In addition even more data can be specified in the file itself or in an external python module.

To access more advanced features users must add the pyblue tag loader into each page: {% verbatim %} {% load pytags %} {% endverbatim %}

{% load pytags %} You can now make use of specific tags for example: {% verbatim %} {% link "context.html" %} {% endverbatim %} will produce the following: {% link "context.html" %}.

Why? The {% verbatim %} {% link "context.html" %} {% endverbatim %} command performs a regular expression search on all files in the directory and once it finds a match it produces a link to it with the proper path. This keeps links correct even after moving files! Note how in this case the file is located in info/context.html

In PyBlue just about every string is used as a regular expression. All a user needs to specify is the shortest unambiguous part of the file.
