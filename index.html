Welcome to Pyblue

A simple static site generator. Zero configuration. Works with any existing site.

PyBlue allows the reuse of data and visuals (header/footer) across many pages and it can generate static sites from dynamic templates.

PyBlue also makes it easy to use Markdown inside of webpages.

This file is located in docs/src/index.html of the PyBlue distribution.

Why PyBlue?

There is no shortage of static html generators. PyBlue is a personal project that aims to address my personal annoyances with other options. I have evaluated quite a few alternatives, some for many months yet slowly each drove me nuts. Hence the motivation to write my own. The following are some of the reasons why PyBlue exists:

  1. Requires no configuration or predetermined directory/file naming. I hate when software starts by imposing rules on what should go where.
  2. Automatic linking! PyBlue keeps track of your links based on the file name. You can move the files around to other directories and these will be found and linked correctly.
  3. Allows users to embed code/files with syntax highlighting.
  4. Allows users to load data into every page. Data can either go into each page as comments or a python module can be loaded that may contain any python object.
  5. Advanced templating. Users may use the Django Templating system and all that it offers.
  6. Bootstrap integration. Handy shortcuts to Bootstrap features.
  7. Tiny codebase. PyBlue is around 500 lines of code. Runs on Python 2.7 and 3.4 as well.

Install

Install it with easy_install pyblue or pip install pyblue --upgrade, or download it from the PyBlue PyPI site.

  • Launch pyblue to serve a directory pyblue serve -r docs/src.
  • View your site by visiting http:://localhost:8080.
  • Edit your pages and make changes. Reload to page to see the effect.
  • Finally, generate static output: pyblue make -r docs/src -o docs/html

Context

The pages are rendered through the Django Template engine. Users automatically get additional context for each of the pages as variables that exists.

For example the {{ page }} variable is present:

  • Writing: {{ page.name }} will produce: Index
  • Writing: {{ page.last_modified }} will produce: Wednesday, December 03, 2014
  • Writing: {{ page.size|filesizeformat }} will produce: 5.3Â KB

Note how in this last case we also made use of the code>filesizeformat builtin filter from Django.

Users may add more information even load a python module into the page.

Links

To access more advanced features users must add the pyblue tag loader into each page: {% load pytags %}. You can now make use of specific tags.

For example {% link "context.html" %} will produce the following: Context.

Why? The {% link "context.html" %} command performs a regular expression search on all files in the directory and once it finds a match it produces a link to it with the proper path. This keeps links correct even after moving files! Note how in this case the file is located in info/context.html

Note that the search string is used as a regular expression! All a user needs to specify is the shortest unambiguous part of the file and PyBlue will find it and link it.

Markdown

To include markdown content place it between markdown template tags.

{% load pytags %}
{# title=Markdown embedded into html template #}

{% markdown %}
Hello World!
============

Now you can write markdown in your templates. This is good because:

* Markdown is awesome
* Markdown is less verbose than writing html by hand
* The page title is: {{ page.title }}

Note how you can use other template constructs inside the markdown section.

Back to home: {% link "index.html" %}

{% endmarkdown %}

Include

To include syntax highlighted code write {% code "context.py" %}

#
# The data file named context.py in the root folder
# will be loaded automatically into the page.
#

NUMBERS = range(5)

# It may contain any valid python construct.

def say_hello(name):
    return "Hello %s" % name

greeting = say_hello("World!")

Now this file is also special because of its name. The contents of it are available in the page. For example writing {{ context.greeting }} produces: Hello World!

Bootstrap

There are helper methods for [Bootstrap][boostrap].

For example {% boostrap_cdn %} will include the CSS and Javascript files used with Boostrap.
