Welcome to PyBlue

A simple static site generator. Zero configuration. Works with any existing site.

PyBlue allows the reuse of data and visuals (header/footer) across many pages and it can generate static sites from dynamic templates.

This page was created with PyBlue from the file located in docs/src/index.html.

Why PyBlue?

There is no shortage of static html generators. PyBlue is a personal project that aims to address my personal annoyances with other options. I have evaluated quite a few alternatives even used some options over longer periods of time. Neither of the choices did everything I needed for my work. Hence came the motivation to write my own static site generator.

The following are some of the reasons why PyBlue exists:

  1. Pyblue requires no initial configuration, special file naming scheme to get started. Pyblue will work with existing html files. Every feature is optional!
  2. Automatic linking! Pyblue can be told to produce correct links automatically. What this means is that one can freely reorganize the site at any time without having to worry about broken links.
  3. Pyblue can embed and syntax highlight code.
  4. Pyblue integrates seamlessly with Markdown.
  5. Pyblue can be told to load data into any page. The template system can operate on this data.
  6. Advanced templating Django Templating system and all that it offers.
  7. Bootstrap integration. Handy shortcuts to Bootstrap features.
  8. Tiny codebase. PyBlue is around 500 lines of code. Runs on Python 2.7 and 3.4 as well.

Install

Install it with easy_install pyblue or pip install pyblue --upgrade, or download it from the PyBlue PyPI site.

Note: Version 2.0 of pyblue moved from Mako Templates to Django Templates and with that introduced other semantic differences. To install the last version of the 1.* line use: easy_install pyblue==1.1.2 or pip install pyblue=1.1.2. See also the previous docs at: Pyblue 1.1.2

Usage

  1. Launch pyblue to serve a directory. Serve these docs: pyblue serve -r docs.
  2. View your site by visiting http:://localhost:8080.
  3. Edit your files and make changes. Reload your page to see the effect.

Once the site is done generate the final static output

  1. pyblue make -r docs -o html
  2. Upload the files to your static server

To see extra help on options run: pyblue -h

Features

Context

The pages are rendered through the Django Template engine. Users automatically get additional context for each of the pages as variables that exists. For example the {{ page }} variable is always available:

  • Writing: {{ page.name }} will produce: Index
  • Writing: {{ page.last_modified }} will produce: Thursday, December 10, 2015
  • Writing: {{ page.size|filesizeformat }} will produce: 7.1Â KB

Note how in this last case we also made use of the filesizeformat builtin filter from Django. Users may add more information even load a python module into the page. More details on the Context page.

Links

To access more advanced features users must add the pyblue tag loader into each page: {% load pytags %}. Once this is added to a page users can make use of pyblue specific tags. For example {% link "context.html" %} will produce the following: Context.

How does that work? The {% link "context.html" %} command performs a regular expression search on all files in the directory and once it finds a match it produces a link to it with the proper path. This keeps links correct even after a file is moved! Note how in this case the file is located in info/context.html

Note that the search string is a regular expression that does not need a full path to the file. It is sufficient to specify the shortest unambiguous part of the filename PyBlue will find it and link it.

Markdown

To include markdown content place it between

{% markdown %}

Hello **Markdown**!

{% endmarkdown %}

template tags.

Fenced code blocks

When using markdown fenced codeblocks with highlighting hints can be used:

```python
for line in open("data.txt"):
    print line.strip()[:10]
```

will render as:

for line in open("data.txt"):
    print line.strip()[:10]

Include code

To include syntax highlighted code from a file write {% code "context.py" hint="python" %}

#
# The data file named context.py in the root folder
# will be loaded automatically into the page.
#

NUMBERS = range(5)

# It may contain any valid python construct.

def say_hello(name):
    return "Hello %s" % name

greeting = say_hello("World!")

Add {% pygments_css %} to the header of the page to generate the CSS classes needed to syntax color embedded code.

This file is also special because of its name: context.py. The contents of this python module are available inside every page under the name of context. For example writing {{ context.greeting }} produces: Hello World!. Hence we can add more complex datastructures into every page.

Include markdown

Alternatively one can instruct PyBlue to include the rendered content of markdown files. For example to include example.md in original form one would use {% code "example.md" %} this would produce:

**Note**: This text comes from a **markdown** file.

This same file could be included as html with {% include_markdown "example.md" %} In that case it will produce the following:

Note: This text comes from a markdown file.

Include other templates

The django templating system allows including other templates or sections.

{% include "sometemplate.html" %}

Bootstrap

There are helper methods for Bootstrap.

For example {% boostrap_cdn %} will include the CSS and Javascript files used with Boostrap.
